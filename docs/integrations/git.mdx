### Setup Instructions

<Steps>
  <Step title="Create GitHub Personal Access Token">
    Visit [GitHub Settings \> Personal access tokens](https://github.com/settings/personal-access-tokens) and click **"Generate new token"**.
  </Step>
  <Step title="Configure Token Permissions">
    Enable the following permissions:

    - **repo**: Full control of private repositories
    - **workflow**: Update GitHub Action workflows
    - **user**: Read user profile information
  </Step>
  <Step title="Connect GitHub Account in CodinIT">
    Navigate to **Settings \> Git \> GitHub** and enter your personal access token.
  </Step>
  <Step title="Authorize Repository Access">
    Grant CodinIT permission to access your repositories when prompted.
  </Step>
  <Step title="Start Using Git Features">
    Import repositories, clone projects, or push changes to GitHub.
  </Step>
</Steps>

</ParamField>

  Your GitHub username. Automatically detected from token if not provided.
</ParamField>

### Setup Instructions

<Steps>
  <Step title="Create GitLab Personal Access Token">
    Go to [GitLab Personal Access Tokens](https://gitlab.com/-/profile/personal_access_tokens) and create a new token.
  </Step>
  <Step title="Configure Token Scopes">
    Enable the following scopes:

    - **api**: Full API access to your projects
    - **write_repository**: Push access to repositories
    - **read_user**: Read your user profile information
  </Step>
  <Step title="Connect GitLab Account in CodinIT">
    Navigate to **Settings \> Git \> GitLab** and enter your access token.
  </Step>
  <Step title="Configure GitLab Instance (Optional)">
    Specify your GitLab instance URL if using GitLab self-hosted.
  </Step>
  <Step title="Start Using Git Features">
    Import projects, clone repositories, or push changes to GitLab.
  </Step>
</Steps>

## Repository Operations

### Cloning Repositories

<CodeGroup>

```bash Clone from GitHub
# Clone a public repository
https://github.com/username/repository-name

# Clone a private repository (requires authentication)
https://github.com/username/private-repository-name

# Clone specific branch
https://github.com/username/repository-name --branch feature-branch
```


```bash Clone from GitLab
# Clone a public project
https://gitlab.com/username/project-name

# Clone a private project (requires authentication)
https://gitlab.com/username/private-project-name

# Clone from self-hosted GitLab
https://gitlab.company.com/username/project-name
```

</CodeGroup>

### Repository Cloning Features

<Expandable title="Smart URL Handling">
  - **SSH to HTTPS conversion**: Automatically converts SSH URLs to HTTPS for browser compatibility
  - **Git suffix removal**: Handles URLs with or without `.git` suffix
  - **Branch specification**: Clone specific branches for targeted development
  - **CORS proxy integration**: Uses built-in proxy for cross-origin Git operations
</Expandable>

<Expandable title="Authentication Methods">
  - **Token-based authentication**: Uses personal access tokens for secure access
  - **Automatic credential storage**: Stores credentials securely for future operations
  - **Multi-platform support**: Works with both GitHub and GitLab authentication
  - **Session persistence**: Maintains authentication across browser sessions
</Expandable>

### Pushing to Repositories

<CodeGroup>

```bash Push to GitHub
# Push current workspace to new GitHub repository
codinit-workspace -> github.com/username/new-repository

# Push to existing repository
workspace-changes -> github.com/username/existing-repository

# Push to specific branch
feature-branch -> github.com/username/repository-name:feature-branch
```


```bash Push to GitLab
# Push current workspace to new GitLab project
codinit-workspace -> gitlab.com/username/new-project

# Push to existing project
workspace-changes -> gitlab.com/username/existing-project

# Push to development branch
dev-branch -> gitlab.com/username/project-name:development
```

</CodeGroup>

  - **Restrict pushes**: Limit who can push directly to protected branches

## Troubleshooting

### Common Git Issues

<AccordionGroup>
  <Accordion title="Authentication Errors">
    **Problem**: Git operations fail with authentication errors.

    **Solutions**:

    - Verify your personal access token is valid and not expired
    - Check token permissions for the target repository
    - Ensure token has appropriate scopes for the operation
    - Try regenerating the token if issues persist
  </Accordion>
  <Accordion title="Repository Access Issues">
    **Problem**: Cannot access private repositories or perform operations.

    **Solutions**:

    - Verify repository permissions and access level
    - Check if the repository exists and is spelled correctly
    - Ensure your token has the required permissions
    - Contact repository administrator for access if needed
  </Accordion>
  <Accordion title="Push/Pull Errors">
    **Problem**: Push or pull operations fail with merge conflicts or errors.

    **Solutions**:

    - Pull latest changes before pushing to avoid conflicts
    - Resolve any merge conflicts locally before pushing
    - Check branch permissions and protection rules
    - Verify remote repository URL is correct
  </Accordion>
  <Accordion title="Large Repository Issues">
    **Problem**: Repository cloning or operations fail with large repositories.

    **Solutions**:

    - Clone specific branches instead of entire repository
    - Use shallow clone for faster initial cloning
    - Check repository size and consider using Git LFS for large files
    - Contact repository maintainer for access optimization
  </Accordion>
</AccordionGroup>
