CodinIT's WebContainer provides a complete Node.js development environment running entirely in your browser. Built on [StackBlitz's WebContainer API](https://webcontainers.io), it eliminates the need for local development setup while providing full Node.js capabilities.

## Core Features

<ParamField type="runtime">
  Complete Node.js environment including npm, yarn, and all standard Node.js APIs running in your browser.

  - **Process Management**: Spawn and manage multiple Node.js processes
  - **File System**: Full read/write access to project files
  - **Network**: HTTP servers, WebSocket connections, and API calls
  - **Environment Variables**: Support for all standard Node.js environment variables
</ParamField>

<ParamField type="package">
  Native support for npm and yarn package management.

  - **Dependency Installation**: Install packages from npm registry
  - **Script Execution**: Run npm scripts and build processes
  - **Package Resolution**: Automatic dependency resolution and caching
  - **Lock File Support**: Compatible with package-lock.json and yarn.lock
</ParamField>

### Preview System

<ParamField type="preview">
  Real-time preview of your applications with hot reload support.

  - **Instant Updates**: See changes reflected immediately in preview
  - **Error Overlay**: Visual error indicators in preview with stack traces
  - **Cross-Origin Support**: Handle COEP requirements for modern web features
  - **Port Management**: Automatic port allocation and management
</ParamField>

<ParamField type="errors">
  Comprehensive error forwarding from preview iframes.

  - **Uncaught Exceptions**: Capture and display JavaScript errors
  - **Promise Rejections**: Handle unhandled promise rejections
  - **Stack Traces**: Detailed error stack traces with source mapping
  - **Error Context**: File path, line numbers, and execution context
</ParamField>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Container Won't Start">
    **Error**: WebContainer fails to initialize

    **Solutions**:

    - Check browser compatibility and enable required permissions
    - Clear browser cache and reload the page
    - Disable browser extensions that might interfere
    - Ensure stable internet connection for initial setup
  </Accordion>
  <Accordion title="Preview Not Loading">
    **Error**: Application preview shows blank screen or errors

    **Solutions**:

    - Check for JavaScript errors in the preview console
    - Verify all dependencies are properly installed
    - Ensure development server is running correctly
    - Check for port conflicts or firewall issues
  </Accordion>
  <Accordion title="Performance Issues">
    **Error**: Slow startup or laggy interface

    **Solutions**:

    - Close unused browser tabs to free up memory
    - Restart the WebContainer session
    - Check for memory leaks in your application
    - Optimize bundle size and remove unused code
  </Accordion>
  <Accordion title="Network Problems">
    **Error**: Package installation or API calls fail

    **Solutions**:

    - Check internet connection stability
    - Verify npm registry accessibility
    - Clear WebContainer cache if corrupted
    - Try using a different package registry mirror
  </Accordion>
</AccordionGroup>
